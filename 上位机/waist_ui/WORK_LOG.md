# 项目工作日志

## 项目概述
康复医疗仪表盘系统 - ESP8266与上位机通信

## 需求沟通记录

### 第一次需求确认
- **通信模式**: TCP客户端（ESP8266作为客户端连接上位机）
- **传感器**: 暂时不需要，后续可能添加压力传感器
- **WiFi配网**: 串口命令方式（WIFI=SSID,PASSWORD）
- **LED指示**: 需要
- **调试输出**: 启用

### 第二次需求确认（关于电机vs传感器）
- **4个通道**: 电机推杆伸缩长度（0-100）
- **压力传感器**: 暂时不考虑
- **控制方式**: 上位机滑动条控制
- **显示**: 人体图 + 4个卡片

### 第三次需求确认（关于界面布局）
- **数据监测界面**: 传感器卡片+人体图
- **力控调节界面**: 保留原样（滑动条+连接状态）
- **通信日志**: 独立界面（日志+命令输入）
- **导航Tab**: 5个（数据监测、通信日志、康复训练、趣味游戏、用户自定义）

## UI设计思路

### 1. 设计理念：数字孪生仪表盘 (Digital Twin Dashboard)
- 放弃原本拥挤的三栏布局
- 采用**非对称双栏布局 (70% 可视化 : 30% 控制区)**
- 亮色模式 (Light Mode)

### 2. 配色方案
- 主色：医疗蓝 (#0078D4)
- 警示色：柔和红 (#C42B1C)
- 背景：灰白 (#F3F3F3)
- 数值大，标签小，视觉层级清晰

### 3. 组件规范（必须遵守）
- 所有文本使用QFluentWidgets字体规范 (`TitleLabel`, `BodyLabel`, `CaptionLabel`)
- 所有图标使用 `FluentIcon`
- 所有按钮使用 `PushButton` / `PrimaryPushButton`
- 所有滑动条使用 `Slider`
- 尽量避免手写原生QSS

## 工作成果

- **文件**: `ESP8266CODE/mian.cpp`

### 4. 通信协议
- **格式**: `[0xAA][长度][类型][数据][校验和][0x55]`
- **命令 (上位机→ESP8266)**:
  - 0x01: 设置力控参数
  - 0x02: 获取状态
  - 0x03: 系统复位
  - 0x04: 参数辨识
