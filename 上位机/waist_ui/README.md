# 康复医疗仪表盘

基于PySide6和QFluentWidgets的康复医疗设备监控上位机系统，支持ESP8266通信和实时数据监测。

## 项目简介

本项目是一个现代化的康复医疗仪表盘系统，用于实时监测和控制康复设备。系统采用模块化设计，界面美观易用，支持与ESP8266设备进行TCP Socket通信，实现力控参数调节、传感器数据实时显示等功能。

### 主要功能

- **数据监测界面**：实时显示四个传感器通道（左肩LF、左膝LB、右肩RF、右膝RB）的数据
- **力控参数调节**：通过滑动条和数字框精确控制各通道的力控参数
- **系统状态监控**：实时显示设备连接状态和通讯状态
- **快捷指令**：支持参数自动辨识和系统复位功能
- **多界面支持**：包含康复训练、趣味游戏、用户自定义等扩展界面

### 技术栈

- **GUI框架**：PySide6 (Qt for Python)
- **UI组件库**：QFluentWidgets
- **通信协议**：TCP Socket
- **编程语言**：Python 3.10+

## 快速开始

### 环境要求

- Python 3.10 或更高版本
- Windows 操作系统（推荐）
- 网络连接（用于ESP8266通信）

### 安装依赖

```bash
pip install PySide6 qfluentwidgets
```

### 运行程序

```bash
cd pyside6
python main.py
```

### 配置ESP8266

在 `config/settings.py` 中修改ESP8266的连接参数：

```python
ESP_HOST = '192.168.1.100'  # ESP8266的IP地址
ESP_PORT = 8080              # ESP8266的端口号
```

## 使用说明

### 界面使用说明

#### 数据监测界面

数据监测界面是系统的主界面，分为左右两个区域：

**左侧区域 - 患者数字孪生区**
- 显示人体模型图片
- 四个状态卡片分别显示各传感器通道的实时数据
- 状态卡片包含：通道名称、数值、进度条、状态徽章

**右侧区域 - 指挥控制中心**

1. **力控参数调节**
   - 四个滑动条分别控制LF、LB、RF、RB四个通道
   - 每个滑动条旁边有数字框，可精确输入数值
   - 滑动条和数字框双向绑定，实时同步

2. **快捷指令**
   - **参数自动辨识**：点击后自动识别设备参数
   - **系统复位**：将所有滑动条和状态卡片重置为0

3. **系统状态**
   - 显示当前系统状态（就绪、通讯正常等）
   - 出现问题时显示重试按钮

### 通信协议说明

#### 数据包格式

```
[起始符][长度][类型][数据][校验和][结束符]
  0xAA    1字节  1字节  N字节   1字节   0x55
```

#### 命令类型

- `0x01` - 设置力控参数
- `0x02` - 获取状态
- `0x03` - 系统复位
- `0x04` - 参数自动辨识

#### 数据类型

- `0x10` - 传感器数据
- `0x11` - 状态数据
- `0x12` - 响应数据

#### 通信流程

**下行数据（控制命令）：**
```
UI → Data Manager → ESP8266 Client → ESP8266
```

**上行数据（传感器数据）：**
```
ESP8266 → ESP8266 Client → Protocol → Sensor Data → Data Manager → UI
```

### 配置说明

#### 修改窗口配置

在 `config/settings.py` 中修改窗口配置：

```python
APP_NAME = '康复医疗仪表盘'
WINDOW_WIDTH = 1100
WINDOW_HEIGHT = 700
MIN_WIDTH = 1000
MIN_HEIGHT = 640
```

#### 修改力控参数范围

在 `config/settings.py` 中修改力控参数配置：

```python
FORCE_MIN = 0
FORCE_MAX = 100
FORCE_DEFAULT = 0
```

#### 修改传感器通道

在 `config/settings.py` 中修改传感器通道配置：

```python
CHANNELS = ['LF', 'LB', 'RF', 'RB']
CHANNEL_NAMES = {
    'LF': '左肩',
    'LB': '左膝',
    'RF': '右肩',
    'RB': '右膝'
}
```

## 开发规范

### 项目结构规范

#### 模块划分原则

- **单一职责**：每个模块只负责一个功能领域
- **高内聚低耦合**：模块内部功能相关，模块间依赖最小
- **可扩展性**：新增功能时只需添加对应模块

#### 文件命名规范

- **模块文件**：使用小写字母和下划线（如：`data_monitor.py`）
- **类名**：使用大驼峰命名（如：`DataMonitorInterface`）
- **私有方法**：使用双下划线前缀（如：`__initWidget`）

#### 目录组织规范

```
模块名/
├── __init__.py          # 模块初始化，导出公共接口
├── 核心功能文件.py      # 主要功能实现
└── 辅助功能文件.py      # 辅助功能实现
```

### 编码规范

#### 命名规范

- **变量名**：小写字母和下划线（如：`status_cards`）
- **函数名**：小写字母和下划线（如：`create_left_widget`）
- **类名**：大驼峰命名（如：`DataMonitorInterface`）
- **常量名**：全大写字母和下划线（如：`FORCE_MAX`）

#### 代码风格

- **缩进**：使用4个空格
- **行宽**：每行不超过120字符
- **注释**：使用中文注释，清晰说明代码意图
- **导入顺序**：标准库 → 第三方库 → 本地模块

#### 文档字符串

- **模块文档**：每个模块文件顶部添加模块说明
- **类文档**：每个类添加类说明
- **方法文档**：公共方法添加方法说明

### 版本控制规范

#### Git提交规范

- **提交信息格式**：`<类型>: <简短描述>`
  - 类型：`feat`（新功能）、`fix`（修复）、`docs`（文档）、`style`（格式）、`refactor`（重构）、`test`（测试）、`chore`（构建）
  - 示例：`feat: 添加ESP8266通信功能`、`fix: 修复滑动条鼠标事件异常`

- **提交粒度**：每个提交只包含一个逻辑变更
- **分支管理**：
  - `main`：主分支，稳定版本
  - `develop`：开发分支，新功能开发
  - `feature/*`：功能分支，单个功能开发

#### 版本号规范

采用语义化版本号：`主版本号.次版本号.修订号`

- **主版本号**：不兼容的API修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正

示例：`1.0.0`、`1.1.0`、`1.1.1`

## 常见问题

### Q: 程序无法连接ESP8266？

A: 请检查以下几点：
1. 确保ESP8266和PC在同一WiFi网络
2. 检查防火墙设置，确保端口未被阻止
3. 确认ESP8266的IP地址和端口配置正确
4. 检查ESP8266是否正常工作

### Q: 滑动条无法拖动？

A: 这是QSS样式问题，已修复。如果仍有问题，请检查 `resource/light/demo.qss` 中是否注释掉了Slider样式的负边距。

### Q: 界面显示异常？

A: 请检查以下几点：
1. 确保已安装所有依赖：`pip install PySide6 qfluentwidgets`
2. 检查资源文件是否完整：`resource/` 目录下应有 `body.png` 和样式文件
3. 尝试删除 `__pycache__` 文件夹后重新运行
